# تدقيق ما قبل إعادة التصميم الموحّدة — المرحلة 0

تاريخ التنفيذ: 2026-07-18. فرع العمل: `redesign/unified-ui-v2`. هذا التدقيق فعلي (قراءة ملفات + بحث نصي + typecheck/build فعليين على الحالة الأساسية قبل أي تعديل)، وليس افتراضًا.

## 0. حالة أساسية موثّقة (baseline قبل أي تغيير)

- `pnpm install` من جذر monorepo: نجح (1422 حزمة، 9.7 ثانية، مخزن مؤقت محلي).
- `pnpm run typecheck`: **نظيف** (بلا أخطاء) على الحالة الأساسية.
- `pnpm run test:regression`: **كل السلاسل نجحت** (seo, identity, guardrails, search-normalize, scholars-integrity, governance, counts, library-integrity, assistant-suggested, sheikhs-dedup, inheritance-engine, learning-paths-engine [53 نجح/0 فشل], learning-assessment-grading [14 نجح/0 فشل], category-tree [14 نجح/0 فشل], quran-data [5 نجح/0 فشل — بما فيها تطابق عدد الآيات 6236 وعدد السور 114]).
- `pnpm run build`: **نجح** بالكامل، شمل: توليد المحتوى، `vite build`، توليد `version.json`، `prerender.mjs` (119 مسارًا مُصيَّرًا مسبقًا، 58 مُستثنى عمدًا)، `post-build-seo` (دمج 727 ملف prerender)، `test-dynamic-404-safety.mjs` (فحص 96 عالِمًا و117 كتابًا — كل سجل حي له prerender مطابق)، `test-no-regressions.mjs` (727 صفحة مفحوصة، لا تسرّب لمحتوى الرئيسية، لا فتاوى، لا ذكر أزهر).
- حجم `dist/` الأساسي: **~25MB** إجمالاً (JS ~7.9MB، CSS ~1.6MB). أكبر حزمة JS: `index-*.js` بحجم 591KB (خام) / 161KB (gzip).

هذه الحالة الأساسية نظيفة تمامًا وتصلح كنقطة انطلاق آمنة — أي فشل لاحق في `typecheck`/`build`/الاختبارات هو من تعديلات هذه المهمة، لا من المستودع الأصلي.

## 1. حجم المشروع الفعلي (لماذا 20 مرحلة كاملة تتجاوز نطاق جلسة واحدة)

- 711 ملف TypeScript/TSX تحت `src/` (431 `.tsx` + 280 `.ts`).
- **222 مسارًا (`<Route>`)** مُسجَّلة في `src/App.tsx` وحده.
- **19 ملف CSS** تحت `src/` بإجمالي **69,693 سطرًا**. الأكبر: `src/styles/elite-2026.css` (31,771 سطرًا)، `src/index.css` (18,711 سطرًا)، `src/styles/design-system.css` (5,304 سطرًا)، `src/styles/majalis-v2.css` (3,073 سطرًا)، `src/styles/modern-2026.css` (1,753 سطرًا)، إضافة لملفات أصغر متخصصة (quran.css, mind-map.css, admin.css...).
- أكثر من **20 فرعًا سابقًا** بنوايا "إعادة تصميم كاملة" موجودة في تاريخ المستودع (راجع سياق المهمة) — لا واحد منها وصل للإنتاج، ما يؤكد أن هذا النطاق (20 مرحلة، منصة حية بمحتوى ديني) يحتاج تنفيذًا تراكميًا موثَّقًا لا دفعة واحدة.

**الخلاصة الصادقة**: هذا ليس تطبيقًا صغيرًا يُعاد تصميمه من الصفر؛ هو منصة إنتاجية ناضجة بخمسة أنظمة CSS متوازية متراكمة من محاولات سابقة (`index.css`, `design-system.css`, `majalis-v2.css`, `elite-2026.css`, `modern-2026.css`) + محتوى ديني حسّاس يمنع أي توليد بالذكاء الاصطناعي. تنفيذ المراحل 1-18 بالكامل بجودة إنتاجية موثّقة (تحقق فعلي بعد كل بند لا ادّعاء) يتطلب أكثر من جلسة تنفيذ واحدة. هذا التدقيق يوثّق الوضع الحقيقي ليكون أساسًا لتنفيذ تراكمي لاحق (commit-per-item، بوابة typecheck/build بعد كل بند) بدل تكرار مصير الفروع الـ20 المهجورة.

## 2. نظام التصميم — ما هو موجود فعليًا مقابل المطلوب

**موجود بالفعل (مفاجأة إيجابية — لا تُعِد بناءه)**:
- `src/index.css` يحتوي فعلاً على `:root` بمتغيرات تصميم شبه موحّدة: سلّم مسافات (`--sp-1` … `--sp-16`)، سلّم طباعة (`--text-xs` … `--text-4xl`)، سلّم حواف (`--radius-sm` … `--radius-pill`)، سلّم z-index، ألوان أساسية زمردية (`--clr-primary: #176B57`، `--clr-primary-deep`, إلخ) مطابقة تمامًا لهوية العلامة المطلوبة (`#0E6E52` / `#153025`).
- `docs/design-tokens.md` و`docs/design-system.md` موجودان مسبقًا ويوثّقان قاعدة صارمة ضد البني/الذهبي المفرط (بالضبط القيد المطلوب رقم 5) — نتيجة تدقيق تباين ألوان سابق ناجح (مذكور في ذاكرة المشروع).
- Safe-area: **مُطبَّق فعليًا وبشكل واسع** — 3 ملفات (`index.css`, `styles/quran.css`, `styles/elite-2026.css`) تحتوي على أكثر من 30 استخدامًا لـ `env(safe-area-inset-*)` تغطي الهيدر، التنقل السفلي، Bottom Sheets، الأزرار العائمة. `viewport-fit=cover` موجود في `index.html`. `@capacitor/status-bar` و`@capacitor/keyboard` مثبَّتان ومُستخدَمان في `src/lib/capacitor-utils.ts`. **المرحلة 2 من المهمة أقرب لتحسين/تدقيق منها لبناء من الصفر.**

**فجوات حقيقية مؤكدة**:
1. **الخط**: التعليق في `index.css` صريح: "خط المنصة الموحَّد **IBM Plex Sans Arabic**" — وليس **Almarai** كما يتطلب القيد رقم 5. Amiri Quran مُحمَّل فعلاً وصحيح للنص القرآني (`index.html` سطر 49). تبديل خط الواجهة كامل المنصة يتطلب تعديل تحميل الخط في `index.html` + كل مرجع CSS لـ`IBM Plex Sans Arabic` (يحتاج مسحًا شاملاً منفصلاً، أثر واسع على 222 مسارًا).
2. **تشتت CSS**: نفس المتغير (مثال: الأخضر الأساسي) مُعرَّف بقيم متفاوتة عبر 5 ملفات مختلفة (`--clr-primary` في index.css، `--ds-emerald`/`--msk-gold` في design-tokens.md/design-system.css). هذا يطابق فخّ "الأخضر على الأخضر" الموثَّق سابقًا في ذاكرة المشروع (`[class$="-hero"]`، `!important` خارجي يهزم inline).
3. **الألوان اليدوية**: **8,213** تكرارًا لقيم hex مباشرة (`#RGB`/`#RRGGBB`) داخل ملفات CSS، و**833** قيمة hex مباشرة داخل ملفات `.tsx` (خارج tokens). القيد "امسح كل الألوان اليدوية خارج tokens" يعني عمليًا مراجعة آلاف المواضع — غير قابل لإنجاز آمن ضمن جلسة واحدة بلا اختبار بصري شامل (كسر تباين سابق مذكور في الذاكرة: `[class$="-hero"]` كسر 96+ صفحة عند تنفيذ مشابه).
4. **الوضع الداكن**: 4 ملفات CSS فقط تشير إلى `prefers-color-scheme`/`dark:` — تغطية جزئية فقط، رغم أن `docs/design-tokens.md` يوثّق ألوان dark mode معتمدة (`#D9AE5C` على `#171C2E`) لم تُطبَّق شموليًا بعد.
5. `theme-color` في `index.html` وفي `src/lib/seo.ts` **ثابت** (`#153025` / `#176B57` حسب المكان) وليس ديناميكيًا حسب الوضع الفاتح/الداكن كما يتطلب القيد.

## 3. البنية — الراوتر والمكونات

- الراوتر: `wouter` (وليس React Router) — 222 `<Route>` في `src/App.tsx` مباشرة (ملف ضخم واحد، لا تقسيم حسب Feature)، مع `lazyWithRetry` للتحميل الكسول لمعظم الصفحات.
- `src/components/BottomNavBar.tsx` موجود (تنقل سفلي واحد بالفعل، لا تكرار ظاهر). `src/components/SiteFooter.tsx` (فوتر موحّد فعليًا بـ5 مجموعات روابط، 100+ رابط داخلي).

- مجلدات مكوّنات مقسّمة حسب Feature بالفعل: `adhan`, `admin`, `assistant`, `circles`, `citation`, `design`, `fawaid`, `fiqh-council`, `home`, `knowledge`, `landmarks`, `layout`, `learning`, `lessons`, `platform`, `prayer`, `qa`, `quiz-game`, `quran`, `rag`, `reading`, `recommendations`, `rulings`, `sheikh`, `seo`, `universities`, `ui`. هذا تنظيم صحي نسبيًا — العمل الحقيقي المطلوب هو توحيد الأنماط البصرية داخلها لا إعادة الهيكلة الكاملة.
