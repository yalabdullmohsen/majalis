## 2026-07-24 — الدفعة 4: iOS/RTL — إصلاحان كوديّان حقيقيّان (أول تعديل تطبيقي هذه الجلسة)

اتّبعت أولوية الميثاق التالية (iOS/RTL). فحص `rg` عن أنماط توليد
مكسورة (`0None`, `undefined`, `NaN`) في CSS كشف عن:

1. **`right/left/top/bottom: 0None` — قيمة CSS غير صالحة، 40 حالة عبر
   ملفين.** بحث `rg -o '[0-9]+None\b'` كشف 32 حالة في `elite-2026.css`
   و8 في `majalis-v2.css`، كلها نفس النمط بالضبط (`.mzd-hero__pattern`
   وأمثاله — تراكب زخرفي مُصمَّم ليغطي كامل الأب عبر `inset:0` لكن
   `0None` قيمة غير صالحة يتجاهلها المتصفح، فلا يغطي التراكب شيئًا
   فعليًا). إصلاح ميكانيكي آمن 100%: `sed -i 's/0None/0/g'` على كلا
   الملفين، تحقّقت لاحقًا صفر بقايا. لا استخدام شرعي لـ"None" كلاحقة
   قيمة CSS في أي مكان آخر بالمشروع (تحقّقت).

2. **حقل بحث الرئيسية وكل حقول النص/textarea/select الموحَّدة تحت
   16px — تُطلق تكبيرًا تلقائيًا غير مقصود في iOS Safari عند التركيز
   (بند صريح في الميثاق §12.9).** القاعدة الفعلية الحاكمة (الأحدث في
   الملف، توازي specificity مع باقي التعريفات المكرَّرة، تفوز بترتيب
   المصدر) هي `input[type="text"], ..., textarea, select` تحت عنوان
   "§77e — نظام الإدخال الموحّد" (سطر ~29287 في `elite-2026.css`) —
   كانت `font-size: 0.9375rem` (15px). تحقّقتُ فعليًا لا توجد قاعدة
   لاحقة في cascade تتجاوزها. رُفعت لـ`1rem` (16px) — تغيير بصري ضئيل
   (+1px)، يُصلح الجذر المشترك لكل حقول الإدخال العامة في الموقع دفعة
   واحدة بدل ملاحقة كل تكرار فرعي (يوجد عدة تعريفات مكرَّرة أقدم بنفس
   المشكلة، لكنها مُتجاوَزة فعليًا بترتيب الـcascade — لم أُعدِّلها، dead
   code في الممارسة).

**لم يُختبَر بصريًا** (بلا متصفح متاح هنا) — التغييران ميكانيكيان
منخفضا المخاطر جدًا (استبدال حرفي لقيمة غير صالحة، وزيادة 1px على
خط)، لكن يُستحسَن تحقّق بصري سريع على `/` (حقل البحث) عند إتاحة الفرصة.

**تحقّق سلبي مهم (لا خطأ — وثِّق لمنع إعادة تحقيق لاحقة):**
`.bottom-nav--v2` في `elite-2026.css` (سطر 13486) ظاهريًا بلا
`safe-area-inset-bottom` ولا حد أدنى touch-target — بدا كخطأ محتمل عند
أول نظرة. **تتبّعت الـcascade كاملًا (main.tsx يحمّل 9 ملفات CSS،
`final-release.css` آخرها):** القاعدة الفعلية الفائزة في
`final-release.css:388` تُطبّق `--safe-bottom: env(safe-area-inset-bottom, 0px)`
بشكل صحيح تمامًا (بما فيها يسار/يمين للـlandscape)، و`.bottom-nav--v2
.bottom-nav__tab { min-width: 44px; min-height: 44px; }` — مطابق تمامًا
لبند الميثاق §12.4. **لا حاجة لإصلاح — السطر الأقدم في elite-2026.css
مُتجاوَز فعليًا (dead code في الممارسة، ليس في الكود).** هذا مثال مباشر
لسبب أهمية بند `css-architecture-consolidation` في backlog.json: كثرة
الملفات المتراكبة تجعل أي "خطأ" ظاهري يحتاج تتبّع cascade كامل قبل
الحكم عليه، لا قراءة قاعدة واحدة بمعزل عن الباقي.
