# قواعد إلزامية دائمة — مشروع المجلس العلمي

## النشر والفروع (الأهم)
- اعمل على فرع main فقط. يُمنع منعا باتًا أي push أو merge أو PR نحو فرع production، ويُمنع تعديل إعدادات Vercel — إلا إذا طلب المستخدم صراحة في رسالته "انشر للإنتاج".
- الإنتاج (www.majlisilm.com) يتبع فرع production فقط؛ main فرع معاينة.

## المحتوى الديني
- يُمنع توليد أي محتوى ديني ذاتيًا. النقل الحرفي من مصادر موثقة معتمدة فقط.
- كل محتوى جديد يدخل بـ is_approved = false و verified_by = null وينتظر المراجعة البشرية.
- النص المخزّن والمعروض — خاصة القرآني — لا يُعدّل أبدًا؛ التطبيع للبحث فقط، والزخرفة عبر CSS/markup فقط.

## الهوية والجودة
- الهوية: emerald #0E6E52 و #153025، أسطح عاجية، خطوط Amiri للقرآن و Almarai، أيقونات Lucide، RTL-first مع WCAG AA.
- لا حذف لأي ميزة قائمة، لا مكونات مكررة، لا حلول مؤقتة (هوامش سالبة/إخفاء عناصر) — أصلح السبب الجذري.
- لا مكتبات جديدة إلا بمبرر مثبت.

## التنفيذ والتسليم
- اختبر الجزء المعدّل قبل اعتباره مكتملًا، ولا تدّعِ نجاحًا دون تحقق عملي.
- سلّم تقريرًا نهائيًا مختصرًا بالعربية في نهاية كل مهمة.
