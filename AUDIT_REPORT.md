### 2. فحص العرض الخادمي (SSR)

- **الوضع السابق:** SPA كامل — كل المسارات تخدم index.html نفسها. index.seo.html كانت صدفة واحدة ثابتة للرئيسية فقط مع `http-equiv="refresh"` يعيد التوجيه فوراً.
- **المشكلة:** محركات البحث التي لا تنفذ JavaScript ترى HTML فارغاً لجميع الصفحات.
- **الحل المنفّذ:** سكريبت `scripts/prerender.mjs` يولّد 123 صدفة HTML ثابتة عند البناء.

### 3. فحص SEO التقني

| العنصر | الوضع |
|---|---|
| sitemap.xml | موجود ومُولَّد ديناميكياً عبر /api/sitemap |
| robots.txt | صحيح، يرفض /admin و /login |
| JSON-LD | موجود في index.seo.html (WebSite + SearchAction) |
| JSON-LD لكل صفحة | يُضاف ديناميكياً عبر applyPageSeo() |
| BreadcrumbList | يُضاف ديناميكياً عبر breadcrumbJsonLd() |
| Canonical | يُضاف ديناميكياً |
| hreflang | غير موجود (الموقع عربي فقط — مقبول) |
| OG tags | موجودة لكن ثابتة قبل الإصلاح |

**ما تم بعد الإصلاح:**
- 123 ملف HTML في dist/ بعناوين وأوصاف وJSON-LD صحيحة لكل مسار
- Breadcrumb ديناميكي لكل مسار
- noscript content مع عنوان الصفحة ووصفها

## ما أُخفي من التنقل ولماذا

لم يُخفَ أي رابط من الروابط الرئيسية الأربع (PRIMARY_NAV_ITEMS)، لأنها تُشير إلى أقسام مكتملة. أما الروابط خلف FeatureGate فهي غير مرئية في NAV_GROUPS أصلاً.

**نقطة تتطلب قرارك:**
- `/quran/surah-stories` يظهر في NAV_GROUPS وHOME_MORE_SECTIONS وMOBILE_MORE_NAV لكنه يعرض شاشة ComingSoon. الخيار: (أ) إخفاؤه إلى حين اكتماله، أو (ب) ربطه بـ SurahStoriesPage الموجود.

## التوصيات المؤجلة (تحتاج قرارك يا يوسف)

1. **غرفة الدراسة والمخزن (study_room, vault):** كلاهما مكتمل تقنياً لكن مغلق بإعلام. هل تودّ فتحهما للمستخدمين؟ يكفي تغيير القيمة إلى `true` في `FEATURES_DEFAULT`.

2. **`/quran/surah-stories`:** يظهر في 3 أماكن بالتنقل ويؤدي إلى شاشة انتظار. هل أُخفيه أم أُسنده إلى `SurahStoriesPage` الموجود؟

3. **SSR على مستوى التفاصيل الديناميكية:** صفحات تفاصيل الدروس (`/lessons/:id`) والمقالات لا تحظى بـ pre-rendering لأن محتواها يأتي من Supabase. الحل: Edge Function أو ISR — يتطلب تعديلاً على Vercel config وإعداداً إضافياً.

4. **تفعيل `learning_calendar` flag:** صفحة التقويم مكتملة تقنياً (`LearningCalendarPage.tsx`) ومفهرسة في seo-routes، فقط الإعلام يمنعها. هل تفعّلها؟

5. **حجم الحزمة الرئيسية (109KB gzip):** تخفيضها يتطلب نقل seed data من الحزمة الأساسية إلى ملفات JSON تُحمَّل عند الطلب. عملية تحتاج مراجعة دقيقة لكل استدعاء seed.

6. **صور WebP/AVIF:** لا تحويل تلقائي للصور. يتطلب إضافة vite-imagetools أو تحويل يدوي للملفات في /public/images.

## خطوات النشر

### Vercel
- لا تغييرات مطلوبة في vercel.json — الملفات الثابتة في dist/ تُخدَم تلقائياً.
- عند كل `git push`، يجري Vercel الـ build command الذي يشمل الآن prerender.mjs.

### Supabase
- لا تغييرات مطلوبة.

### Google Search Console
- بعد النشر: اطلب فهرسة `/start-here` و`/lessons` و`/quran` و`/seerah` من GSC.
- راجع تقرير AMP/Page Experience للتأكد من أن الصفحات الجديدة مُفهرسة بشكل صحيح.
- أرسل `sitemap.xml` مجدداً إلى GSC بعد هذا النشر.
