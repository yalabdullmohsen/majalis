# سير عمل النشر — المجلس العلمي

## المبدأ الأساسي

**لا يصل أي تغيير إلى الموقع الحي `majlisilm.com` إلا بموافقة يدوية صريحة من المالك.**

---

## فروع المشروع

| الفرع | الغرض | Vercel |
|---|---|---|
| `main` | التطوير الرئيسي | Preview فقط |
| `feat/*`, `fix/*`, `dev/*` | فروع تطوير فرعية | Preview فقط |
| **`production`** | **الإنتاج الحي** | **نشر للموقع الحي** |

---

## سير العمل اليومي

### 1. التطوير (Preview)
```
جميع التطوير يتم على: main أو فروع فرعية
كل push → Vercel ينشر Preview URL (ليس الموقع الحي)
```

### 2. النشر للإنتاج (يدوي)
```
1. تأكد من أن main مستقر وجاهز للنشر
2. افتح Pull Request: main → production
3. راجع التغييرات في رابط Preview
4. وافق على PR بنفسك ثم ادمجه
5. Vercel ينشر production تلقائياً عند الدمج
```

---

## قواعد الحماية (مُطبَّقة تلقائياً)

فرع `production` محمي في GitHub:
- ❌ لا يمكن الدفع المباشر (`git push origin production`)
- ✅ النشر فقط عبر Pull Request تُوافق عليه أنت
- ❌ لا يمكن حذف الفرع أو force push

---

## خطوة يدوية متبقية في Vercel

> **مطلوبة منك مرة واحدة فقط** لتفعيل النظام كاملاً

### تغيير فرع الإنتاج من `main` إلى `production`:

1. اذهب إلى: **vercel.com/dashboard → مشروع majalis-majalis**
2. اضغط **Settings** (الإعدادات)
3. اضغط **Git** من القائمة الجانبية
4. ابحث عن **Production Branch**
5. غيّر القيمة من `main` إلى `production`
6. احفظ التغيير (Save)

**بعد هذه الخطوة:**
- أي push على `main` = Preview فقط
- أي دمج في `production` = نشر على `majlisilm.com`

---

## الحالة الراهنة

- ✅ فرع `production` أُنشئ من آخر commit منشور على الإنتاج
- ✅ حماية GitHub مُفعَّلة (require PR + لا force push + لا حذف)
- ⏳ تغيير فرع الإنتاج في Vercel **(يدوي — راجع الخطوة أعلاه)**
