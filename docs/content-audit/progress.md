## أول عنصر للبدء منه عند الاستئناف
انتهت جولة الفحوصات **البرمجية** المنهجية (تكرار، تواريخ، أعداد، بقايا ميزات، SEO) عبر كل ملفات seed الرئيسية — نظيفة أو مُصحَّحة بالكامل. المتبقي هو **`b010`**: مراجعة **علمية/شرعية يدوية** عميقة سجلاً-سجلاً لبقية الأقسام (علوم القرآن، التفسير، التجويد، الحديث ومصطلحه، الفقه وأصوله، العقيدة، السيرة الكاملة، الصحابة، التاريخ الإسلامي، الأخلاق، اللغة العربية) — لا يمكن أتمتتها، تحتاج قراءة فعلية لكل نص والتحقق من دقته الشرعية/التاريخية بنفس منهجية إصلاح اليسع/ذي الكفل/إدريس. ابدأ بقسم واحد فقط في كل دفعة (مقترح: الصحابة، أو مصطلح الحديث). `b011`، `b014`، `b018` معلَّقة (needs_source) — لا تُعاد فتحها آلياً بلا مصدر جديد.

## قيود التنفيذ المتّبعة هذه الجلسة
- Agent واحد فقط، بلا subagents.
- `pnpm run build` الكامل يتطلب `PORT=3000 BASE_PATH=/` بيئياً (حزم تسويقية غير مرتبطة تتطلبه إلزامياً حتى للبناء الثابت).
- عند تشغيل أي سكربت مزامنة (`sync-seo-data.ts` وغيره): **راجع `git diff --stat` دائماً** قبل الإضافة للـcommit وأعد للحالة الأصلية أي ملف غير مرتبط بالدفعة الحالية (`git checkout --`) — ثبت عملياً أنها قد تُنتج انحرافاً واسعاً (b006) أو تكشف ملفات مولَّدة قديمة غير مرتبطة (rulings-encyclopedia chunks, seo-prerender/fiqh-council/*) لا يجب تضمينها.
- لا دمج ولا push إلى main — فقط `automation/content`.
