# نظام الألوان المعتمد — مجالس

**مُحدَّث 2026-07-24** — هذا الملف كان يوثّق لوحة قديمة (`#0E6E52`) سبق
إلغاؤها فعليًا في الكود بإعادة تصميم الهوية v3 (`#173D35`)، دون تحديث
الملف نفسه. المصدر الوحيد الموثوق للتوكِنز هو
`src/styles/modern-2026.css` (`:root` + `html:not([data-theme="dark"])`)؛
هذا الملف ملخّص منه فقط — عند أي تعارض تُرجَّح قيمة CSS الفعلية.

النقي كما افترضت نسخة سابقة من هذا الملف) — راجع التعليق التفصيلي
أعلى `:root` في `modern-2026.css` لكل قيمة.

## ممنوع منعاً باتاً

- ❌ أي قيمة hex بنية قديمة من لوحة ما قبل v3: `#0E6E52`, `#0A5040`, `#92400E`, `#B45309`, `#a16207`, `#c8a24a`
- ❌ فئات Tailwind: `amber-*`, `orange-8*`, `orange-9*`, `yellow-7*`, `yellow-8*`, `yellow-9*`
- ❌ كتابة قيمة hex حرفية للأخضر الأساسي داخل مكوّن جديد — استخدم `var(--m26-green-800)` أو الاسم الدلالي المناسب دومًا، لا `#173D35` مباشرة (رُصد 1164 تكرارًا حرفيًا لهذه القيمة وحدها خارج ملفات التوكِن — انظر `docs/ui-audit/design-system.md`)

## مسموح استثنائياً

- ✅ `--m26-gold-500` (`#B89452`) كتفصيل ذهبي رملي هادئ محدود — ليس للنص الأساسي ولا لكل بطاقة
- ✅ الأحمر/البرتقالي/الأخضر فقط عبر `--txt-error` / `--txt-warning` / `--txt-success` الدلالية، لا مباشرة
- ✅ ألوان الوضع المظلم `html[data-theme="dark"]` لها قواعدها الخاصة

## الوضع المظلم

يُطبَّق عبر `html[data-theme="dark"]` بقواعده الخاصة (منفصلة عن `html:not([data-theme="dark"])` أعلاه) — راجع القسم المخصص له في `modern-2026.css` قبل أي تعديل، ولا تفترض قيم الوضع النهاري صالحة فيه.
