# تشغيل موقع "مجالس العلم" الحقيقي — خطوة بخطوة

> هذا مشروع Next.js حقيقي يتصل بقاعدة بياناتك في Supabase.
> اتبع الخطوات بالترتيب. إن تعثّرت، أعطِ المجلد لأي مبرمج وسيشغّله في دقائق.

---

## الخطوة 1: احصل على المفتاح العام من Supabase

1. افتح مشروعك في Supabase.
2. اذهب إلى **Settings** (الترس) → **API**.
3. ستجد:
   - **Project URL** ← هذا موجود عندك: `https://ngmvmlulzacrlicuagyp.supabase.co`
   - **Project API keys** → انسخ مفتاح **`anon` / `public`** (يسمّى أحيانًا Publishable key).

⚠️ لا تنسخ مفتاح `service_role` أبدًا — هذا سرّي ويتجاوز كل الحماية.

---

## الخطوة 2: ضع المفتاح في الموقع

1. في مجلد المشروع، ستجد ملفًا اسمه **`.env.local.example`**.
2. انسخه وأعد تسميته إلى **`.env.local`** (احذف `.example`).
3. افتحه، وضع مفتاحك مكان `ضع_مفتاحك_هنا`:

```
NEXT_PUBLIC_SUPABASE_URL=https://ngmvmlulzacrlicuagyp.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ... (مفتاحك العام الذي نسخته)
```

4. احفظ الملف.

---

## الخطوة 3: ثبّت وشغّل (على جهازك)

1. ثبّت **Node.js** من nodejs.org (نسخة LTS) إن لم يكن مثبتًا.
2. افتح Terminal داخل مجلد المشروع، واكتب:

```bash
npm install
npm run dev
```

3. انتظر حتى ترى رسالة مثل `Local: http://localhost:3000`.
4. افتح المتصفح على **http://localhost:3000** — موقعك يعمل ويقرأ من قاعدة بياناتك الحقيقية! ✅

---

## الخطوة 4: تأكّد أن كل شيء متصل

- صفحة **المشايخ** → يُفترض أن ترى مشايخك الستة (من القاعدة).
- صفحة **الفوائد** → يُفترض أن ترى فوائدك (من القاعدة).
- أنشئ حسابًا من **دخول** → جرّب التسجيل في درس.
- ادخل بحساب المشرف → ستظهر **لوحة التحكم**.

> إن ظهرت الصفحات فارغة، تأكد أنك شغّلت ملفي SQL في Supabase (`01_schema` و `02_seed`).

---

## الخطوة 5: ارفعه على الإنترنت (Vercel)

1. ارفع المجلد إلى GitHub (استخدم GitHub Desktop إن لم تكن تعرف Git).
2. افتح **vercel.com** → سجّل بحساب GitHub.
3. **Add New Project** → اختر مستودعك.
4. في **Environment Variables**، أضف نفس المتغيرين من `.env.local`.
5. اضغط **Deploy**.

✅ بعد دقيقتين، موقعك على الإنترنت برابط حقيقي يفتحه أي شخص.

## ما الذي يعمل الآن فعليًا؟

| الصفحة | المصدر | تفاعلي؟ |
|---|---|---|
| الرئيسية | — | روابط الأقسام |
| الدروس | جدول `lessons` | ✅ تسجيل/إلغاء يُحفظ في حسابك |
| المشايخ | جدول `sheikhs` | ✅ صفحة كاملة لكل شيخ + دروسه |
| المكتبة | جدول `library_items` | ✅ فلترة حسب النوع |
| الإعجاز العلمي | جدول `scientific_miracles` | ✅ فلترة قرآن/سنة |
| الفوائد | جدول `fawaid` | ✅ إرسال + مراجعة |
| لوحة التحكم | كل الجداول | ✅ محمية للمشرف + إضافة درس + مراجعة فوائد |
| الدخول | مصادقة Supabase | ✅ تسجيل/دخول حقيقي |

## ملاحظات مهمة

- **المكتبة والإعجاز فارغان حاليًا** — لأنك لم تضف محتوى فيهما بعد. أضف من لوحة التحكم أو مباشرة في جداول Supabase. (الإعجاز العلمي يحتاج محتوى موثّقًا — لا تضف نظريات غير ثابتة.)
- **التصميم مبسّط بأنماط مكتوبة يدويًا** (بدون Tailwind) لتسهيل التشغيل عليك دون إعداد إضافي. يمكن لاحقًا نقل واجهات نموذجك الأصلي الأجمل (القرآن، العبادات، الخريطة) إلى هذا المشروع.

## الأقسام التي لم تُنقل بعد من نموذجك الأصلي

هذه موجودة في نموذجك التجريبي وتحتاج نقلًا لاحقًا (كل منها يحتاج جدولًا أو منطقًا إضافيًا):
المساجد (الخريطة) · القرآن (المصحف) · العبادات (المواقيت والقبلة) · التسميع · المجمع الفقهي.

أخبرني أيها تريد نقله أولًا، أو نضيف ميزة جديدة (نقاط، إشعارات، بحث موحّد...).
