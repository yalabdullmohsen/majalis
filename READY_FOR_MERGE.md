## إصلاح إضافي (2026-07-18، commit ae3d40e9): ضعف التعرّف الصوتي

المالك أبلغ أن التعرّف على الكلمات "ضعيف جدًا". السبب الجذري الفعلي لم يكن
دقة Google الصوتية نفسها، بل خلل برمجي: كان الكود يُغذّي محرك المحاذاة
بنتائج **مؤقتة غير مستقرة** (interim) من Web Speech API كأنها كلمات
مؤكَّدة، فتولّد أخطاء وهمية كثيرة أثناء "تنقيح" Chrome لتخمينه. أُصلح
عبر معالجة نتائج `isFinal=true` فقط + إعادة تشغيل تلقائية عند توقّف
Chrome الصامت للاستماع المستمر. تحقّق حي (محاكاة أحداث Web Speech API
حقيقية داخل متصفح فعلي) أكَّد الإصلاحين. **لا يرفع هذا دقة محرك Web
Speech API العام (غير المتخصص بالقرآن) إلى 100% فعليًا** — يُصلح خللاً
كان يُفسد تعرّفًا صحيحًا أصلاً؛ دقة قريبة من الكمال تحتاج مزوّدًا خادميًا
متخصصًا لا يزال غير موصول (`NOT_CONFIGURED`).
